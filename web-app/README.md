# 港口联邦学习演示前端

基于 **React + TypeScript + Vite + Tailwind CSS** 的可视化演示项目，面向港口集装箱场景展示多模型联邦学习的端到端体验。每个模型模块都由独立的 JSON mock 数据驱动，方便根据现场故事快速调整指标、任务和推理素材。

## 快速开始

```bash
cd web-app
npm install
npm run dev    # http://localhost:5173
npm run build  # 产物位于 dist/
```

## 主要特性

- **多模型集群首页**：港口垂类大模型、自动抓/放箱控制、视觉辅助、行人安全等 7 个卡片，展示本地 vs 联邦版本差异、节点覆盖与健康状态。
- **模型详情套件**：概览、训练表现、联邦任务、推理体验四大页签，可在本地/联邦版本之间即时切换并查看对应指标、版本时间线与数据集概览。
- **联邦任务看板**：每个模型拥有独立的运行中终端日志、排队任务（含优先级/排队信息）与已完成任务（合并港口、指标增益、评测按钮），并提供快速提交新任务的表单。
- **推理体验**：支持在不同模型版本间切换、选择预置图像/视频样本、查看检测数量/置信度/时延，且保留上传自定义样本的入口。
- **Mock 数据体系**：`src/data/models/*.json` 描述各模型的指标、曲线、任务、推理素材；`modelService` 负责统一读取，可替换为真实 REST/WebSocket 或 Flower/FedML SDK 接口。

## 二次开发建议

1. 将 `modelService` 中的静态数据替换为真实后端调用（REST、GraphQL 或 WebSocket），或通过 Flower/FedML/NVIDIA FLARE 的调度接口注入实时指标。
2. 在推理区域接入真实检测结果渲染（Canvas/SVG 叠框），并与历史检测记录联动。
3. 扩展任务看板：加入节点贡献度热力图、资源占用监控或任务事件时间线，提升可讲述性。
4. 使用 `src/data/models/*.json` 继续新增模型模块，只需复制模板并更新指标即可立刻呈现在首页与详情视图中。
